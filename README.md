# Как использовать компактный индекс NormaCS API

## Что вы получили

- **`normacs_api_compact_index.md`** — компактный индекс всей документации NormaCS API (2 библиотеки типов, 76 объектов/коллекций, 260 методов, 463 свойства, 10 событий, 30 перечислений). ~26 КБ / ~8–10 тыс. токенов.
- Он содержит: инструкцию для LLM, соглашение об именах файлов, шпаргалку по типовым задачам и полный реестр имён.
- Он **не содержит** сигнатур, параметров, типов возврата и примеров — их LLM подгружает на лету.

## Схема работы «роутер + ленивая загрузка»

```
┌──────────────────────────────────────────────┐
│         СИСТЕМНЫЙ ПРОМПТ                     │
│   normacs_api_compact_index.md (~9K токенов) │
└──────────────────────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────┐
│ Запрос пользователя:                         │
│ "Напиши VBA-скрипт, который подключается     │
│  к NormaCS и ищет все документы со словом    │
│  'бетон' в тексте"                           │
└──────────────────────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────┐
│ LLM по индексу определяет, какие страницы    │
│ нужны:                                       │
│   NormaCSAPI~Application~Connect.md          │
│   NormaCSAPI~Application~StartDocument...    │
│   NormaCSAPI~Application~WaitForEvent.md     │
│   NormaCSAPI~Find~Documents.md               │
│   NormaCSAPI~Document.md                     │
│   NormaCSAPI~Enumerations~NormaCSEventType.. │
└──────────────────────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────┐
│ LLM читает эти файлы (через инструмент       │
│ доступа к папке / WebFetch / вы вставляете)  │
│ и генерирует корректный код.                 │
└──────────────────────────────────────────────┘
```

## Способы подключения

### Вариант A. Claude в Cowork / Claude Code с локальной папкой

1. Положите папку с документацией (со всеми `.md` файлами) рядом или внутри рабочей папки Claude.
2. В системный промпт (или в начало чата) вставьте содержимое `normacs_api_compact_index.md`.
3. Добавьте одну строку вида: «Документация лежит в `/path/to/normacs_docs/`. Когда тебе нужны подробности по конкретному методу/свойству, читай нужный `.md` файл инструментом Read.»
4. Задавайте вопросы — Claude сам запросит нужные страницы.

### Вариант B. Claude API (Anthropic SDK)

Паттерн: **system prompt = компактный индекс**, а загрузку страниц реализуйте как *tool*:

```python
import anthropic, pathlib

DOC_DIR = pathlib.Path("./normacs_docs")
SYSTEM = pathlib.Path("normacs_api_compact_index.md").read_text(encoding="utf-8")

tools = [{
    "name": "read_api_doc",
    "description": "Читает страницу документации NormaCS API по имени файла "
                   "(например 'NormaCSAPI~Application~Connect.md'). Имя файла "
                   "берётся из компактного индекса в системном промпте.",
    "input_schema": {
        "type": "object",
        "properties": {"file": {"type": "string"}},
        "required": ["file"]
    }
}]

def read_api_doc(file: str) -> str:
    p = DOC_DIR / file
    if not p.exists():
        return f"ERROR: файл {file} не найден"
    return p.read_text(encoding="utf-8")

client = anthropic.Anthropic()
messages = [{"role": "user", "content": "Напиши C#-код, который подключается к NormaCS и ищет ГОСТ 27751."}]

while True:
    resp = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=4096,
        system=SYSTEM,
        tools=tools,
        messages=messages,
    )
    messages.append({"role": "assistant", "content": resp.content})
    if resp.stop_reason != "tool_use":
        print(resp.content[-1].text)
        break
    tool_results = []
    for block in resp.content:
        if block.type == "tool_use":
            tool_results.append({
                "type": "tool_result",
                "tool_use_id": block.id,
                "content": read_api_doc(block.input["file"]),
            })
    messages.append({"role": "user", "content": tool_results})
```

### Вариант C. ChatGPT / любой другой LLM без инструментов

1. Вставляете `normacs_api_compact_index.md` как начало чата (system / первое сообщение).
2. Когда LLM отвечает «мне нужны файлы X, Y, Z», вручную копируете содержимое этих `.md` и вставляете следующим сообщением.

### Вариант D. Собственный RAG

1. Компактный индекс — **всегда** в контексте (это «каталог»).
2. Детальные `.md` файлы проиндексированы в векторной БД.
3. Когда модель упоминает имя файла — ретривер подмешивает его в контекст в обход векторного поиска (exact-match выигрывает).

## Почему не дать LLM весь 63-КБ индекс?

Можно, но:

- На длинных диалогах модель начинает «расплываться», путая похожие имена (`Find.Documents` vs `Application.Find.Documents`).
- Компактный индекс + ленивая загрузка даёт *более точные* ответы, потому что для кодогенерации модель видит настоящие сигнатуры, а не догадывается по имени.
- Токены дешевле: вы платите за 9K постоянно и за 1–3K дополнительно на нужные страницы, а не за 63K в каждом запросе.

## Проверка качества ответов

Индекс построен из вашего `index.md` без потерь (подсчёт совпал с плоскими cross-ref списками: 260 методов, 463 свойства, 10 событий). Если LLM «вспоминает» метод, которого нет в индексе, — это галлюцинация, в индексе это видно мгновенно по Ctrl+F.

## Рекомендация по языку кода

Имена API одинаковые во всех языках. Самые удобные варианты:

1. **C# / .NET через COM Interop** — лучше всего для новых проектов (типизация, современный тулинг). Нужно `Add Reference → COM → NormaCS API Type Library`.
2. **Delphi / Pascal** — «родной» для NormaCS, есть готовые примеры в их поставке.
3. **VBA (AutoCAD/Excel/Word)** — для интеграции с офисными/CAD-документами.
4. **Python через `pywin32`** — быстрые скрипты и автоматизация; хуже с типами.

Просто скажите Claude в запросе, какой язык нужен.
